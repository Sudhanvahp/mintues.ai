# Minutes AI

AI-powered meeting recorder that transcribes audio, identifies speakers, and generates structured notes — runs in any browser, no installation needed for users.

---

## Features

- **One-tap recording** — browser-based mic capture, no app install
- **Auto transcription** — Groq Whisper large-v3 with timestamps
- **Speaker diarization** — identifies who said what (Speaker 1, Speaker 2…)
- **AI meeting notes** — structured markdown notes generated by Llama 3.3 70B
- **Smart title** — auto-generates a 2–5 word title from the transcript
- **Chat with recording** — ask questions about any meeting
- **Audio playback** — in-app player with speed control (1x / 1.5x / 2x) and ±15s skip
- **Export** — download notes or transcript as text
- **Share** — generate a public link to share with anyone
- **Rename / tag / rate** — organise your recordings

---

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | Next.js 14 (App Router), Tailwind CSS |
| Backend | Python FastAPI + uvicorn |
| Transcription | Groq API — Whisper large-v3 |
| AI Notes + Chat | Groq API — Llama 3.3 70B Versatile |
| Speaker Diarization | pyannote-audio 3.3.2 (runs locally) |
| Database | SQLite + SQLAlchemy |
| Audio Recording | Browser AudioContext + ScriptProcessorNode → WAV |

**Total cost: $0/month** — Groq free tier (14,400 req/day), pyannote runs locally after one-time model download.

## Setup

### 1. Get API Keys (both free)

**Groq API Key**
1. Go to [console.groq.com](https://console.groq.com)
2. Sign up → API Keys → Create API Key
3. Copy the `gsk_...` key

**HuggingFace Token** (for pyannote model download — one-time only)
1. Go to [huggingface.co](https://huggingface.co) → sign up
2. Accept the model license at [pyannote/speaker-diarization-3.1](https://huggingface.co/pyannote/speaker-diarization-3.1)
3. Settings → Access Tokens → New token → copy `hf_...` key

---

### 2. Backend Setup

```bash
cd backend
pip install -r requirements.txt
```

Create `backend/.env`:
```
GROQ_API_KEY=gsk_your_key_here
PYANNOTE_TOKEN=hf_your_token_here
FRONTEND_URL=http://localhost:3000
```

Start the backend:
```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

---

### 3. Frontend Setup

```bash
cd frontend
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)

## How It Works

```
Browser mic → AudioContext → PCM chunks → encodeWAV() → WAV file
                                                              ↓
                                              POST /recordings/upload
                                                              ↓
                                                  Background pipeline:
                                                  1. Get audio duration
                                                  2. Groq Whisper → segments
                                                  3. pyannote → speaker timeline
                                                  4. Merge segments + speakers
                                                  5. Groq Llama → notes + title
                                                              ↓
                                                  Saved to SQLite (status: done)
                                                              ↓
                                              Frontend polls /status every 2s
                                                              ↓
                                              Notes + Transcript + Chat displayed
```

> **Why WAV?** Chrome's MediaRecorder produces streaming webm with broken EBML headers that pyannote cannot parse. Recording as raw PCM → WAV solves this completely.

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/recordings/upload` | Upload audio, start processing |
| `GET` | `/recordings` | List all recordings (`?q=` to search) |
| `GET` | `/recordings/{id}` | Full recording data |
| `GET` | `/recordings/{id}/status` | `{status, step, percent}` |
| `GET` | `/recordings/{id}/audio` | Stream audio file |
| `PATCH` | `/recordings/{id}` | Update title / icon / tags / rating |
| `DELETE` | `/recordings/{id}` | Delete recording + audio file |
| `POST` | `/recordings/{id}/share` | Toggle public share link |
| `POST` | `/recordings/{id}/chat` | Chat about the recording |
| `GET` | `/recordings/{id}/export` | Export (`?format=text_notes\|text_transcript\|audio`) |

## Sharing With Your Team

**Local network (same WiFi):**
Share `http://YOUR_LOCAL_IP:3000` — find your IP with `ipconfig` (Windows) or `ifconfig` (Mac/Linux).

**Public internet (ngrok):**
```bash
# Terminal 1
ngrok http 3000

# Terminal 2
ngrok http 8000
```
After running ngrok, create `frontend/.env.local` with your ngrok backend URL:
```
NEXT_PUBLIC_API_URL=https://your-backend-id.ngrok-free.app
```
Also update `FRONTEND_URL` in `backend/.env` with your ngrok frontend URL, then restart both servers.

## Troubleshooting

| Issue | Fix |
|---|---|
| "No audio captured" alert | Check Windows mic permissions — Settings → Privacy → Microphone. HDMI monitors have no mic. |
| All speakers shown as "Speaker 1" | Make sure audio is being recorded as WAV (check `useAudioRecorder.ts`) |
| Backend 404 on startup | Run uvicorn from inside the `backend/` folder, not the project root |
| pyannote slow | Expected — ~3–5 min processing per 5-min meeting on CPU. Normal behaviour. |
| Groq rate limit | Free tier: 14,400 req/day. Unlikely to hit for normal team usage. |
